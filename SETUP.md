# セットアップガイド（GitHub Pages公開 ＋ Google連携）

このアプリを「Claudeアカウント不要で夫婦どちらの端末からも使える」＋「Gmail自動取得」「給与明細の写真OCR」まで含めてフル活用するための、最初の一回だけの設定手順です。すべて無料でできます。

所要時間の目安：20〜30分（Googleの操作に慣れていなくても大丈夫です）

---

## ステップ0：全体の流れ

1. GitHubにアプリを公開する（誰でもURLだけで開けるようになる）
2. Google Cloudで「Gmailを読む許可」の仕組みを作る
3. Google AI Studioで「写真を読み取るAI」のキーを取得する
4. アプリの設定画面にキーを貼り付ける
5. これまでの家計データ（過去分）を読み込む
6. （任意）Firebaseで夫婦間のリアルタイム同期を設定する

---

## ステップ1：GitHubリポジトリを作成してアプリを公開する

1. https://github.com/ にアクセスし、アカウントがなければ作成（無料）
2. 右上の「+」→「New repository」
   - Repository name: `kakeibo-app`（何でもOK）
   - Public を選択（GitHub Pagesを無料で使うには公開リポジトリである必要があります。中身に実際の金額データは含まれていないので安全です）
   - 「Create repository」をクリック
3. できたリポジトリのページに表示されるコマンドは使わず、この`家計簿アプリ`フォルダで以下を実行してください（PowerShellなどのターミナルで、`家計簿アプリ`フォルダに移動してから）：

```
git remote add origin https://github.com/【あなたのユーザー名】/kakeibo-app.git
git branch -M main
git push -u origin main
```

初回はブラウザでGitHubへのログインを求められることがあります。指示に従ってログインしてください。

4. GitHubのリポジトリページで「Settings」→ 左メニュー「Pages」
   - Source: 「Deploy from a branch」
   - Branch: `main` / `/(root)` を選択して「Save」
5. 1〜2分待つと、ページ上部に公開URLが表示されます：
   ```
   https://【あなたのユーザー名】.github.io/kakeibo-app/
   ```
   **このURLが、これから夫婦で使う本番のアプリのURLです。** このURLはブックマークし、美鈴さんにも共有してください（Claudeアカウントは不要です）。

---

## ステップ2：Google CloudでOAuthクライアントIDを作る（Gmail自動取得用）

1. https://console.cloud.google.com/ にアクセス（Googleアカウントでログイン）
2. 上部の「プロジェクトを選択」→「新しいプロジェクト」→ 名前を`kakeibo-app`などにして作成
3. 左上のメニュー（≡）→「APIとサービス」→「ライブラリ」
   - 検索欄に `Gmail API` と入力 → 表示されたものをクリック →「有効にする」
4. 「APIとサービス」→「OAuth同意画面」
   - User Type: 「外部」を選んで「作成」
   - アプリ名：`わが家の家計簿`、ユーザーサポートメール・デベロッパーの連絡先：ご自身のGmailアドレス
   - スコープの画面はそのまま「保存して次へ」でOK
   - **テストユーザー**の画面で「+ ADD USERS」をクリックし、**夫婦それぞれのGmailアドレスを両方とも追加**（これが一番重要なステップです。ここに追加された人だけがGmail連携を使えます）
   - 最後まで進めて保存。**「アプリを公開」ボタンは押さないでください**（Testing状態のままでOK。Googleの審査が不要になり、すぐ使えます。制限として、約7日ごとに再ログインが必要になりますが、家庭用アプリとしては問題ありません）
5. 「APIとサービス」→「認証情報」→「+ 認証情報を作成」→「OAuthクライアントID」
   - アプリケーションの種類：「ウェブ アプリケーション」
   - 名前：任意（例：kakeibo-web）
   - 「承認済みの JavaScript 生成元」に **ステップ1で得たURLのオリジン部分だけ**を追加：
     ```
     https://【あなたのユーザー名】.github.io
     ```
     （末尾に `/kakeibo-app/` は付けません。スキームとホストだけです）
   - 「作成」をクリックすると、**クライアントID**（`〜.apps.googleusercontent.com`という形式の文字列）が表示されます。これをコピーしてください。

---

## ステップ3：Google AI StudioでGemini APIキーを取得する（写真OCR用）

1. https://aistudio.google.com/apikey にアクセス（Googleアカウントでログイン）
2. 「APIキーを作成」をクリック
3. 表示されたキー（`AIza`で始まる文字列）をコピー

無料枠の範囲で、月に何十枚も明細を読み取れます。

---

## ステップ4：アプリに設定する

1. ステップ1で作った公開URL（`https://…github.io/kakeibo-app/`）をブラウザで開く
2. 「設定」タブ →「🔌 API連携」カード
   - Gemini APIキー：ステップ3のキーを貼り付け
   - Google OAuthクライアントID：ステップ2のクライアントIDを貼り付け
   - 「保存」をクリック
3. 「取込」→「楽天ペイ メール」タブで「🔌 Gmailに接続して取得」を押すと、Googleのログイン画面が開きます。ステップ2でテストユーザーに登録したご自身のGoogleアカウントでログイン・許可してください。

美鈴さんが使う場合も同じURLを開き、美鈴さんご自身のGoogleアカウント（テストユーザーに追加済み）でログインすれば、同じように使えます。

---

## ステップ5：これまでの家計データを読み込む

このフォルダに `kakeibo-backup-history.json` というファイルがあります。これは2023年4月〜2026年5月分の実際の収入・支出・資産データが入った**あなた専用の非公開バックアップ**です（GitHubには含めていません。誰にも共有しないでください）。

1. 公開URLでアプリを開く（初回はデータが空の状態です）
2. 「設定」→「💾 バックアップ」→「📥 読み込む」
3. `家計簿アプリ` フォルダ内の `kakeibo-backup-history.json` を選択

これで、これまでの実績データがすべて反映されます。以降は書き出し（📤）でこまめにバックアップを取ることをおすすめします。

## ステップ6（任意）：Firebaseで夫婦間のリアルタイム同期を設定する

このステップを行うと、**拳さんが入力した内容が自動的に美鈴さんの画面にも反映**されるようになります（今までは書き出し・読み込みでの手動同期でした）。無料でできます。

### 6-1. Firebaseプロジェクトを作る

1. https://console.firebase.google.com/ にアクセス（Googleアカウントでログイン）
2. 「プロジェクトを作成」→ 名前を`kakeibo-app`などにして作成（Googleアナリティクスは「無効にする」でOK）
3. 作成できたら、プロジェクトのトップ画面で「</> （ウェブ）」アイコンをクリック
4. アプリのニックネームを入力（何でもOK）→「アプリを登録」
5. 表示される`const firebaseConfig = {...}`という**コード全体をコピー**しておく（あとで使います）

### 6-2. Firestore（データベース）を有効にする

1. 左メニュー「構築」→「Firestore Database」→「データベースを作成」
2. ロケーションは `asia-northeast1`（東京）など、お好きな場所を選択
3. セキュリティルールは「本番環境モード」を選択して作成
4. 作成後、上部タブ「ルール」を開き、以下の内容に**丸ごと置き換えて**「公開」：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /families/{familyId} {
      allow read, write: if request.auth != null &&
        request.auth.token.email in ['夫のGmailアドレス@gmail.com', '妻のGmailアドレス@gmail.com'];
    }
  }
}
```
　→ **`夫のGmailアドレス@gmail.com`と`妻のGmailアドレス@gmail.com`の部分を、実際にログインする2人のGoogleアカウントのメールアドレスに書き換えてください。** これにより、この2人以外は絶対にデータを読み書きできなくなります。

### 6-3. ログイン方法（Google）を有効にする

1. 左メニュー「構築」→「Authentication」→「Sign-in method」タブ
2. 「追加のプロバイダ」から「Google」を選択 → 有効にする → 保存
3. 同じ画面の「設定」タブ →「承認済みドメイン」に、ステップ1で作ったGitHub PagesのURLのドメイン部分を追加：
   ```
   【あなたのユーザー名】.github.io
   ```

### 6-4. アプリに設定する

1. 公開URLでアプリを開く →「設定」→「☁️ 夫婦間リアルタイム同期」カード
2. 「Firebase設定」の欄に、6-1でコピーした`firebaseConfig`のコード全体をそのまま貼り付け →「設定を保存」
3. 「Googleでログインして同期を開始」をクリックし、6-2のルールに登録したご自身のGoogleアカウントでログイン
4. 美鈴さんの端末でも同じURLを開き、同じ手順（2〜3）を行う（`firebaseConfig`の貼り付けはコピーして共有してもらえばOK。これは秘密の情報ではありません）

これで、どちらかが支出や収入を登録すると、数秒以内にもう一方の画面にも自動で反映されるようになります。

## よくある質問

**Q. Claude内のプレビュー（このセッションのアーティファクトURL）ではGmail連携や写真OCRのボタンが動きません**
A. 想定どおりです。Claudeのプレビュー画面はセキュリティ上、外部サイトとの通信を許可していません。ステップ1で作ったGitHub PagesのURLから開いてください。

**Q. 「このアプリは確認されていません」という警告が出る**
A. OAuth同意画面がTesting状態のため出る、正常な警告です。「詳細」→「（アプリ名）に移動（安全ではないページ）」を選べば進めます。自分たちで作ったアプリなので問題ありません。

**Q. 美鈴さんがログインしようとしたら「アクセスをブロック」と出た**
A. ステップ2の「テストユーザー」に美鈴さんのGmailアドレスを追加し忘れている可能性があります。Google Cloud Console →「OAuth同意画面」→「テストユーザー」から追加してください。

**Q. データは2人でリアルタイムに共有されますか？**
A. ステップ6（Firebase）を設定すればリアルタイムで共有されます。設定していない場合は、各端末のブラウザ内に個別保存され、「設定→データを書き出す／読み込む」での手動同期になります。

**Q. Firebase同期をオンにしたら、片方の古いデータでもう片方の新しいデータが消えてしまわないか心配**
A. 更新日時を比較して新しい方を採用する仕組みになっていますが、念のため**ステップ6を始める前に「設定→データを書き出す」で両方の端末のバックアップを取っておく**ことをおすすめします。初回接続時に想定と違う内容になった場合も、書き出しておいたファイルから「読み込む」で復元できます。
